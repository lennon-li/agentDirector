# Audit Review Checklist

Use this after Gemini or another reviewer audits a task.

Check whether the audit includes:

- approve / revise / reject
- evidence reviewed
- blockers
- risks
- missing validation
- scope concerns
- specific required revisions
- next recommended agent
- next suggested instruction
- saved audit file or handoff update

If the audit says “no risks,” be skeptical. Ask whether:
- assumptions were checked
- data frequency was checked
- saved model formats were checked
- existing helper functions were reused correctly
- tests actually ran
